# AGENTS.md — Portfolio

## Stack
- **Astro 4** (static output, `output: 'static'`), **TypeScript** (strict), **Tailwind 3**, **MDX** content, **Swiper** for sliders
- Deployed to **Netlify**: `npm run build` → `dist/` (configured in `netlify.toml`)

## Commands
| Command | Purpose |
|---------|---------|
| `npm run dev` | Start dev server |
| `npm run build` | Build to `dist/` |
| `npm run preview` | Preview production build |

No lint, typecheck, or test commands exist.

## Project structure
- `src/content/projects/*.mdx` — each file is a project with typed frontmatter (see `src/content/config.ts`)
- `src/content/config.ts` — defines the `projects` collection schema (title, client, url, year, category, tags, accent, cover, gallery, gallery_slider, challenge, solution, result, technologies, featured, order)
- `src/components/` — Astro components only (no JSX/TSX)
- `src/pages/` — static routes: `index.astro`, `portfolio.astro`, `servicios.astro`, `brief.astro`, `gracias.astro`, `projects/[slug].astro`
- `src/layouts/Layout.astro` — master layout with SEO meta, OG tags, JSON-LD (Person, WebSite, FAQPage), ViewTransitions, scroll animations
- `netlify/functions/lead-email.mjs` — serverless function that sends lead notifications via Resend (env: `RESEND_API_KEY`)

## Layout props
`Layout.astro` accepts optional props:
- `title` (string, default: generic site title)
- `description` (string, default: generic site description)
- `imageAlt` (string, default: generic alt) — sets `og:image:alt`
- `keywords` (string, default: generic list) — sets `<meta name="keywords">`

All pages should pass page-specific `imageAlt` and `keywords` when available.

## Key conventions
- **Language**: Spanish (`es_PE`), content and UI copy
- **Design tokens**: `dark` (#151414), `light` (#F8F6F0), `ink` (#1A1A1A), `accent-blue` (#3031E4), `accent-orange` (#FE4201) — defined in `tailwind.config.mjs`
- **Token discipline**: NEVER use arbitrary hex values (e.g. `#ff4108`, `#111111`, `#f2f2ef`) in class attributes or CSS. ALWAYS use the Tailwind config tokens (`bg-ink`, `text-accent-orange`, `border-ink/10`, `bg-light`, etc.). The only exception is the servicios hero dark gradient and its blob overlays, which are brand-specific.
- **Fonts**: `Inter` (sans), `Barlow Condensed` (display). No other font families (e.g. Manrope) should be used in pages — all pages inherit `font-sans` (Inter) from the Layout body.
- **Accessibility**: Skip-to-content link rendered in Layout (`sr-only focus:not-sr-only`), targets `#main-content`. Every page MUST set `id="main-content"` on its `<main>` element.
- **Animation**: `.scroll-animate` class + IntersectionObserver in Layout; `.services-swiper` for mobile horizontal scroll; `ViewTransitions` for SPA navigation
- **Swiper**: Loaded as npm dependency (`swiper`), imported via `import Swiper from 'swiper'` in `[slug].astro` script. CSS loaded via `<style is:global>@import 'swiper/swiper-bundle.css'</style>` in the same component (must be `is:global` to avoid Astro scoping on dynamically created elements like pagination bullets). Init functions called directly at module scope (not waiting for `DOMContentLoaded`) and on `astro:after-swap`.
- **Project page nav**: sticky navbar (`sticky top-0 z-50 bg-light/90 backdrop-blur-md`) with logo + breadcrumb (Home / Portfolio / Title) at top of page, outside `<main>`.
- **Swiper pagination**: styled `#93C5FD` (inactive), `#2563EB` (active), 10×10px bullets, `position: relative` with `margin-top: 1.5rem` below slides.
- **Image format**: WebP preferred; project images in `public/projects/`
- **TypeScript**: path alias `@/*` → `src/*`

## Forms
1. **Contact form** (`Footer.astro`): posts to Netlify Forms (`form-name: portfolio-contact`) with secondary Resend notification via `/netlify/functions/lead-email`
2. **Brief form** (`brief.astro`): 7-step wizard that POSTs to formsubmit.co + opens WhatsApp. Stored entirely client-side (no server endpoint)
3. **Servicios form** (`servicios.astro`): posts to Netlify Forms (`form-name: servicios`) with spinner animation on submit + Resend notification. Error state shows contact fallback.

## SEO / Crawling
- `robots.txt` disallows `/brief` and `/gracias`; allows AI search/citation bots, blocks training crawlers (CCBot, Bytespider)
- `sitemap.xml` auto-generated by Astro at build time
- `<meta robots="index, follow">` on all pages
- `<link rel="canonical">` using `siteUrl + Astro.url.pathname`
- `og:image:alt` set per-page via Layout `imageAlt` prop
- `public/llms.txt` — AI-readable site map for LLM crawlers
- JSON-LD structured data: `Person` + `WebSite` (Layout, all pages), plus page-specific schemas:
  - **/servicios**: `CollectionPage`, `BreadcrumbList`, `ProfessionalService` (with OfferCatalog), `FAQPage`, `HowTo` (with image + estimatedCost), `SiteNavigationElement`
  - **/portfolio**: `CollectionPage`, `BreadcrumbList`, `ItemList` (auto-generated from project collection)
  - **/[slug]**: `BreadcrumbList`, `Article` or `CreativeWork` per project

## Tips
- Adding a project: create a `.mdx` file in `src/content/projects/` with the schema fields, then place images in `public/projects/<slug>/`
- **Image auto-detection**: place images in `public/projects/<slug>/` using naming conventions and the system recognizes them automatically:
  - `cover.webp` — portada / hero (used as thumbnail in grids too)
  - `slider-01.webp`, `slider-02.webp`, … — Swiper carousel
  - `gallery-01.webp`, `gallery-02.webp`, … — masonry gallery
  - Frontmatter fields `cover`, `gallery`, `gallery_slider` override auto-detection when present
- The site has no typecheck, lint, or test step — verify manually with `npm run dev` and `npm run build`
